# Plans (Backend Migration)

This folder contains the step-by-step plan to move the project from “frontend talks to Supabase directly” to a proper backend that owns the business logic and data access.

## Index

1. `01-goals-and-success.md` — goals, non-goals, and “done” definition
2. `02-current-state-inventory.md` — what exists today (and where)
3. `03-target-architecture.md` — the intended end state (BFF + DB)
4. `04-api-contract.md` — endpoints, payloads, errors, and cookies
5. `05-data-model-and-db-plan.md` — tables, constraints, RLS, migrations
6. `06-security-and-secrets.md` — keys, cookies, CORS, and threat model
7. `07-implementation-steps.md` — the execution plan (ordered steps)
8. `08-parity-testing.md` — how to guarantee “no missing shots”
9. `09-cutover-and-deployment.md` — rollout plan + rollback
